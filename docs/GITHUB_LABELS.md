## üìÇ Categories

### üî∂ Type (`type:*`)
Describes what the issue or PR is about.

| Label | Meaning |
|--------|----------|
| `type: bug` | Something is broken. |
| `type: feature` | New feature to add. |
| `type: enhancement` | Improve an existing part. |
| `type: refactor` | Code cleanup without behavior change. |
| `type: documentation` | Docs-related changes. |
| `type: test` | Testing or coverage work. |
| `type: build` | Build/config updates. |
| `type: release` | Preparing a release. |
| `type: security` | Security issue or fix. |

## üß≠ Label Rules
- Use **one `type:`** label per issue/PR.  
- Combine with `status:` and `priority:` when relevant.  
- Use `good first issue` only with clear context.  
- Unsure? ‚Üí Use `status: discussion`.
